template<std::invocable< vk::CommandBuffer > F>
requires std::is_void_v<std::invoke_result_t<F, vk::CommandBuffer>>
void vku::executeSingleCommand ( vk::Device device,
vk::CommandPool commandPool,
vk::Queue queue,
F && f,
vk::Fence fence = {} )
export

Allocate a command buffer from commandPool, record commands, and submit it to queue.

The allocated command buffer will automatically begin before and ended after the command buffer recording. If fence is null handle, no explicit synchronization is performed and caller has the responsibility to synchronize the command buffer execution ends (such like queue.waitIdle()). Otherwise, the function will wait until the command buffer execution ends.

Template Parameters
FFunction that accepts the command buffer and records commands to it.
Parameters
deviceVulkan device. Must be same device from commandPool.
commandPoolCommand pool to allocate a single command buffer.
queueQueue to submit the allocated command buffer.
fFunction that accepts the command buffer and records commands to it.
fenceFence to signal when the command buffer is completed, if presented.
